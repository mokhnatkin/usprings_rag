# Парсинг документации 1С:ИТС (its.1c.ru)

Как программно выгружать инструкции по 1С:ERP с портала 1С:ИТС для пополнения
базы знаний RAG. Проверено на базе `erp25ltsdoc` («1С:ERP Управление
предприятием 2, редакция 2.5 LTS») 2026-07-13.

## Исходные данные

- База документации: https://its.1c.ru/db/erp25ltsdoc
- Учётные данные: `1C_portal_credentials.txt` (в этой папке; в git не попадает,
  добавлен в `.gitignore`). Подписка ИТС активна, доступ к полным текстам есть.
- Тестовый скрипт: `its_parse_test.py` — авторизуется и выгружает одну
  инструкцию в markdown. Пример результата: `docs/manuals/parsed/its_test_1c_kassa_settings.md`.

Запуск теста:

```
uv run --with requests --with beautifulsoup4 python its_parse_test.py out.md
```

## Авторизация (CAS через login.1c.ru)

Портал использует SSO-сервис login.1c.ru (Apereo CAS). Браузер не нужен,
достаточно `requests.Session`. Порядок критичен:

1. **GET любой страницы its.1c.ru** — сервер выдаёт cookie `PHPSESSID`.
   Без этого шага авторизация не привязывается к сессии и контент остаётся
   обрезанным (маркер `paywall` в HTML).
2. **GET формы логина**:
   `https://login.1c.ru/login?service=https%3A%2F%2Fits.1c.ru%2Flogin%2F%3Faction%3Daftercheck%26provider%3Dlogin`
   Из HTML забрать скрытое поле `execution` формы `#loginForm`.
3. **POST на тот же URL** с полями `username`, `password`, `execution`,
   `_eventId=submit`, `rememberMe=on`.
4. CAS редиректит на `its.1c.ru/login/?action=aftercheck&provider=login&ticket=ST-...` —
   сервер валидирует тикет и помечает `PHPSESSID` как авторизованный.
   Признак успеха: в конечном URL есть `ticket=`, на страницах появляется
   «Выход» и исчезает класс `paywall`.

Дальше все запросы в той же сессии отдают полный контент.

## Структура базы erp25ltsdoc

- **Оглавление** — главная страница `https://its.1c.ru/db/erp25ltsdoc`
  (кодировка Windows-1251). Содержит ~3641 ссылку на все разделы вида:
  `/db/erp25ltsdoc/bookmark/<раздел>/<подраздел>`
  Regex для сбора: `href="(/db/erp25ltsdoc/bookmark/[^"]+)"`.
  Рекурсивный краулинг не нужен — полный список разделов лежит в одной странице.
- **Страница раздела** (bookmark) — обвязка портала. Тело документа грузится
  в iframe `id="w_metadata_doc_frame"`, его `src` указывает на чистый HTML:
  `/db/content/erp25ltsdoc/src/<имя файла>.htm` (имя кириллическое, при запросе
  URL-кодировать; в `src` может быть якорь `#...` — отрезать).
- **Тело документа** — чистый HTML только с текстом инструкции: заголовки
  h1–h4, абзацы, списки, таблицы, скриншоты (относительные ссылки в том же
  каталоге `src/`). Несколько соседних подразделов bookmark могут указывать на
  один и тот же htm-файл (страница = глава, якоря = подразделы) — при полном
  обходе дедуплицировать по `src`.

## План полного парсинга (когда понадобится)

1. Авторизоваться (см. выше).
2. Скачать оглавление, собрать все bookmark-ссылки.
3. Для каждой: получить `src` iframe, скачать htm, дедуплицировать по `src`.
4. Извлечь текст (см. `extract_text` в скрипте), сохранить в
   `docs/manuals/parsed/` (markdown, один файл на документ, с URL источника).
5. Пауза ~1 сек между запросами: сайт за DDoS-Guard, агрессивный обход может
   привести к бану. ~3641 страница ≈ 1 час.

## Ограничения и примечания

- Контент защищён авторским правом фирмы «1С», доступ — по платной подписке.
  Использование во внутренней базе знаний для сотрудников — нормальный
  сценарий; публиковать наружу нельзя.
- Скриншоты не индексируем (решение MVP: только текст). При необходимости
  их можно скачивать по относительным ссылкам из того же каталога `src/`.
- Кодировка ответов Windows-1251 — `requests` определяет её сам, вручную
  декодировать не нужно.
- Другие базы ИТС (например, документация к другим конфигурациям) устроены
  так же — меняется только идентификатор базы в URL (`erp25ltsdoc`).
