# Парсинг документации 1С:ИТС (its.1c.ru)

Как программно выгружать инструкции по 1С:ERP с портала 1С:ИТС для пополнения
базы знаний RAG. Проверено на базе `erp25ltsdoc` («1С:ERP Управление
предприятием 2, редакция 2.5 LTS») 2026-07-13.

Два проверенных варианта выгрузки:

- **Markdown (только текст)** — `its_parse_test.py`. Быстро (~2,5 с/инструкция),
  но для индексации нужен md-парсер в ingest (пока не реализован).
- **PDF (текст + картинки)** — `its_parse_pdf_test.py`. Медленнее
  (~26 с/инструкция, почти всё время - скачивание скриншотов), зато результат
  индексируется текущим ingest без доработок кода и содержит скриншоты
  для человека.

## Исходные данные

- База документации: https://its.1c.ru/db/erp25ltsdoc
- Учётные данные: `1C_portal_credentials.txt` (в этой папке; в git не попадает,
  добавлен в `.gitignore`). Подписка ИТС активна, доступ к полным текстам есть.
- `its_parse_test.py` — выгрузка одной инструкции в markdown.
  Пример результата: `docs/manuals/parsed/its_test_1c_kassa_settings.md`.
- `its_parse_pdf_test.py` — выгрузка той же инструкции с картинками и сборка
  в PDF (нужны pandoc и typst в PATH).
- `build_registry.py` / `registry.md` — реестр всех разделов базы (номер,
  название, URL, статусы и таймстемпы скачивания и формирования PDF).
  По нему отслеживается прогресс полного парсинга. Нюанс: оглавление
  запрашивается с дефолтным UA requests — браузерному User-Agent сервер
  отдаёт оболочку с JS-подгрузкой дерева без ссылок.
- `its_parse_all.py` — пакетный парсер всех разделов по реестру
  (см. «Полный парсинг» ниже).

Запуск тестов:

```
uv run --with requests --with beautifulsoup4 python its_parse_test.py out.md
uv run --with requests --with beautifulsoup4 python its_parse_pdf_test.py <папка_вывода>
```

## Авторизация (CAS через login.1c.ru)

Портал использует SSO-сервис login.1c.ru (Apereo CAS). Браузер не нужен,
достаточно `requests.Session`. Порядок критичен:

1. **GET любой страницы its.1c.ru** — сервер выдаёт cookie `PHPSESSID`.
   Без этого шага авторизация не привязывается к сессии и контент остаётся
   обрезанным (маркер `paywall` в HTML).
2. **GET формы логина**:
   `https://login.1c.ru/login?service=https%3A%2F%2Fits.1c.ru%2Flogin%2F%3Faction%3Daftercheck%26provider%3Dlogin`
   Из HTML забрать скрытое поле `execution` формы `#loginForm`.
3. **POST на тот же URL** с полями `username`, `password`, `execution`,
   `_eventId=submit`, `rememberMe=on`.
4. CAS редиректит на `its.1c.ru/login/?action=aftercheck&provider=login&ticket=ST-...` —
   сервер валидирует тикет и помечает `PHPSESSID` как авторизованный.
   Признак успеха: в конечном URL есть `ticket=`, на страницах появляется
   «Выход» и исчезает класс `paywall`.

Дальше все запросы в той же сессии отдают полный контент.

## Структура базы erp25ltsdoc

- **Оглавление** — главная страница `https://its.1c.ru/db/erp25ltsdoc`
  (кодировка Windows-1251). Содержит 3645 ссылок на все разделы вида:
  `/db/erp25ltsdoc/bookmark/<раздел>/<подраздел>`
  Regex для сбора: `href="(/db/erp25ltsdoc/bookmark/[^"]+)"`.
  Рекурсивный краулинг не нужен — полный список разделов лежит в одной странице.
- **Страница раздела** (bookmark) — обвязка портала. Тело документа грузится
  в iframe `id="w_metadata_doc_frame"`, его `src` указывает на чистый HTML:
  `/db/content/erp25ltsdoc/src/<имя файла>.htm` (имя кириллическое, при запросе
  URL-кодировать; в `src` может быть якорь `#...` — отрезать).
- **Тело документа** — чистый HTML только с текстом инструкции: заголовки
  h1–h4, абзацы, списки, таблицы, скриншоты (относительные ссылки в том же
  каталоге `src/`). Несколько соседних подразделов bookmark могут указывать на
  один и тот же htm-файл (страница = глава, якоря = подразделы) — при полном
  обходе дедуплицировать по `src`.

## Вариант PDF: текст + картинки

`its_parse_pdf_test.py` дополнительно к тексту скачивает все `<img>` документа
(ссылки относительные, тот же каталог `src/`), переписывает пути на локальные
и собирает PDF: `pandoc instruction.html -o instruction.pdf --pdf-engine=typst
-V mainfont="Times New Roman" --include-in-header=breakable.typ`.

**Обязательный нюанс верстки**: pandoc оборачивает HTML-таблицы в typst-блок
`#figure(...)`, который по умолчанию неразрывный - многостраничная таблица
отрисовывается поверх текста (наложение строк на границах страниц). Лечится
строкой в заголовке документа: `#show figure: set block(breakable: true)`
(в скрипте - файл `breakable.typ`, подключается через `--include-in-header`).

Замеры на тестовой инструкции (14 скриншотов, 1,7 МБ, PDF 22 страницы):

| Фаза | Время |
|-|-|
| Логин (разово на сессию) | ~12 с |
| HTML инструкции | ~2 с |
| 14 картинок | ~23 с |
| pandoc+typst -> PDF | ~1 с |

Итого ~26 с на инструкцию (без учёта разового логина); узкое место - сеть
(~1,6 с на картинку, последовательно). Полная база (3645 разделов) ≈ 26 часов
последовательно; при необходимости ускоряется параллельной загрузкой картинок
(3-4 потока, осторожно с DDoS-Guard) и дедупликацией повторяющихся скриншотов.

## Полный парсинг: `its_parse_all.py`

Пакетный парсер всех разделов по `registry.md` (вариант PDF с картинками).
Готовые PDF складываются в `docs/manuals/parsed/` (по одному на документ,
имя = имя htm-файла источника) — оттуда их индексирует текущий ingest.

```
cd docs/1C_parsing
uv run --with requests --with beautifulsoup4 python its_parse_all.py [--limit N]
```

Как устроен:

- **Прогресс в реестре**: статусы (`ok` / `err: причина` / `дубль`) и
  таймстемпы пишутся в `registry.md` после каждого раздела (атомарная запись).
  Прерванный прогон продолжается с места остановки; строки с `err` при
  повторном запуске обрабатываются заново.
- **Ретраи**: каждый HTTP-запрос - до 3 попыток с паузами 5 и 30 с;
  недокачанная картинка пропускается (в лог), документ не срывается.
  Протухшая сессия (paywall) - автоматический перелогин.
- **Дедупликация**: несколько подразделов bookmark часто указывают на один
  htm-файл (глава = один документ, подразделы = якоря) - такие строки
  помечаются `дубль`, PDF не пересобирается. Уникальных документов заметно
  меньше 3645 - реальный объём прогона станет виден по ходу.
- **Паузы** ~0,7 с между запросами (DDoS-Guard); `--limit N` - пробный прогон
  на N разделов.
- Рабочие файлы (HTML + картинки до сборки PDF) - в `docs/1C_parsing/work/`
  (в `.gitignore`).

## Ограничения и примечания

- Контент защищён авторским правом фирмы «1С», доступ — по платной подписке.
  Использование во внутренней базе знаний для сотрудников — нормальный
  сценарий; публиковать наружу нельзя.
- Скриншоты не индексируем (решение MVP: только текстовый слой). В PDF-варианте
  они сохраняются в файл для человека, но в базу знаний попадает только текст.
- Кодировка ответов Windows-1251 — `requests` определяет её сам, вручную
  декодировать не нужно.
- Другие базы ИТС (например, документация к другим конфигурациям) устроены
  так же — меняется только идентификатор базы в URL (`erp25ltsdoc`).
