  - Схема: `documents.archived_at`/`archived_by` (soft-delete) + таблица `index_job`
    (очередь фоновой индексации со статусом); модели `IndexJob`/`IndexJobStatus`,
    миграция 0008. `alembic check` чист.

    документы вне выдачи (H4).
  - Индексация: `indexing/jobs.py` (enqueue, `claim_next` с `FOR UPDATE SKIP LOCKED`,
    `mark_done`/`mark_error`, `reset_stale`) + `indexing/worker.py` (`IndexWorker` -
    фоновый поток в процессе приложения, одна задача за раз, переиспользует прогретую
    BGE-m3). Старт воркера в lifespan (`INDEX_WORKER_ENABLED`), при старте зависшие
    `running` -> `error`.
  - Админка документов: `admin/documents.py` (список со статусом/датой/числом чанков,
    архивация/возврат); эндпоинты `GET /admin/documents` (страница), `GET
    /api/admin/documents`, `GET /api/admin/jobs`, `POST /api/admin/documents/upload`
    (PDF -> папка коллекции + `index_job`), `POST .../{id}/archive|unarchive`.
    Зависимость `require_admin` (403 не-админам), per-collection - `check_collection_access(need_admin=True)`.
  - UI: `templates/admin/documents.html` (селектор коллекции, загрузка с опросом
    статуса задачи, таблица документов с кнопкой архив/возврат); ссылка «Документы»
    в шапке главной, видима админам через новый `GET /api/me`.
- Проверка приёмки:
  - Тесты (`test_soft_delete`, `test_indexing`, `test_admin_documents`), всего 75:
    архивный документ вне выдачи и возвращается; job проходит queued->running->done,
    привязка к документу; ошибка ingest и отсутствие файла -> `error` с текстом;
    `reset_stale` сбрасывает `running`; не-админ -> 403; админ видит только свои
    коллекции; чужая коллекция при архивации/загрузке -> 403; не-PDF -> 422; загрузка
    создаёт `queued`-задачу и кладёт файл.
  - Живой прогон на реальном PDF и BGE-m3 (scratchpad): очередь -> `process_job`
    (ingest ~64с, 1 чанк) -> `done` с document_id -> поиск находит документ по его
    тексту -> архивация исключает из выдачи. Всё вычищено (документ, чанки, задача, файл).
  - `node --check` - app.js и inline-скрипт `admin/documents.html` синтаксически ок.
- Заметки:
  - `index_job.document_id` - FK `ON DELETE SET NULL`: переиндексация файла удаляет
    старый документ, задача не должна ронять удаление.
  - Загрузка идемпотентна по `source_path` (ingest): повторный файл того же имени
    обновляет документ, а не плодит.
  - Живой браузерный проход UI не гоняли (см. «Незакрытые хвосты»).
